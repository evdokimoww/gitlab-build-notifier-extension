# Changelog

Формат вдохновлён [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/); версии по [Semantic Versioning](https://semver.org/lang/ru/).

## [1.4.3] — 2026-06-04

### Изменено

- Вкладки сессий: более заметное выделение активной (фон, рамка, тень).
- При открытии страницы автоматически выбирается первая доступная сессия (лог и образ).

## [1.4.2] — 2026-06-04

### Исправлено

- Переключение вкладок сессий: лог и build-образ завершённых MR больше не пропадают.

## [1.4.1] — 2026-06-04

### Исправлено

- После обновления страницы Promote MR больше не подставляется последний завершённый MR: активная сессия сбрасывается по окончании, форма и лог пустые (история — во вкладках).

## [1.4.0] — 2026-06-04

### Добавлено

- Несколько MR в одну develop-ветку в одной сессии (список по строкам); проверка конфликта перед merge и системное уведомление при конфликте.

## [1.3.0] — 2026-06-04

### Добавлено

- Параллельные сессии promote: несколько MR одновременно, переключение по вкладкам, отмена и закрытие по отдельности.

## [1.2.1] — 2026-06-04

### Исправлено

- Долгие сборки (30+ мин): heartbeat при опросе pipeline, keepalive-alarm в service worker, порог «прерван» увеличен до 10 мин; в логе раз в 5 мин — статус ожидания.

## [1.2.0] — 2026-06-04

### Добавлено

- Promote MR выполняется в service worker: лог и статус в `chrome.storage`, после обновления страницы подхватывается текущий процесс.

## [1.1.4] — 2026-06-04

### Добавлено

- Promote MR: если у MR или production-ветки нет активного pipeline, запуск через API (`merge_requests/…/pipelines` или pipeline на ref).

## [1.1.3] — 2026-06-04

### Изменено

- Promote MR: увеличен лог, доп. опции в аккордеоне, кнопка «Новый MR» после получения build-образа.

## [1.1.2] — 2026-06-04

### Исправлено

- Убран inline-скрипт на `promote.html` (нарушал CSP расширения `script-src 'self'`).

## [1.1.1] — 2026-06-04

### Исправлено

- Страница Promote MR не реагировала на «Запустить»: в `gitlab-api.js` дублировалось имя `body` в `gitlabFetch` (синтаксическая ошибка, модули не загружались).

## [1.1.0] — 2026-06-04

### Добавлено

- Страница **Promote MR** (`promote.html`): merge feature → develop, promote MR в master/main, ожидание pipeline, вывод build-образа (логика из `gitlab-promote-mr`).
- Расширен `gitlab-api.js` методами merge/create MR, pipeline, артефакты и ветки.

## [1.0.8] — 2026-05-06

### Исправлено

- Мигание favicon во вкладке использует статусные иконки **`notify-ok.png`** и **`notify-fail.png`**: success показывает зелёную иконку с галочкой, fail — красную иконку.

### Удалено

- Логика цветной полоски и мигания внутри страницы GitLab; осталось только изменение заголовка и favicon вкладки.

## [1.0.7] — 2026-05-06

### Исправлено

- Мигание favicon во вкладке больше не чередуется с родным favicon страницы GitLab.

## [1.0.6] — 2026-05-06

### Изменено

- Иконка приложения **`extension-*.png`** заменена на GitLab tanuki-логотип на прозрачном фоне (16 / 32 / 48 / 128 / 512 px).

## [1.0.5] — 2026-05-06

### Изменено

- Иконка **`notify-ok.png`** для уведомлений об успехе: как у fail — круг **#0d8050**, светлая обводка, мягкая тень, **прозрачный** фон; внутри белая галочка.

## [1.0.4] — 2026-05-06

### Изменено

- Иконка **`notify-fail.png`** для системных уведомлений: круг с светлой обводкой и мягкой тенью на **прозрачном** фоне (256×256, RGBA).

## [1.0.3] — 2026-05-06

### Добавлено

- Отдельная **иконка расширения** для панели и менеджера расширений (PNG 16 / 32 / 48 / 128 px; исходник 512 px в `icons/extension-512.png`). Уведомления по-прежнему используют `notify-ok.png` / `notify-fail.png`.

## [1.0.2] — 2026-05-06

### Удалено

- Бейдж на иконке расширения (`V` / `X`); при установке и запуске браузера бейдж сбрасывается.

## [1.0.1] — 2026-05-06

### Исправлено

- Уведомления (и связанный звук, префикс заголовка вкладки, favicon и бейдж) больше **не срабатывают при первом заходе** на страницу merge request или пайплайна, если нужный stage уже успел завершиться. Учитываются только **переходы стадии на открытой вкладке**: с «ещё не завершена» (`empty` / `running`) на завершённое состояние (`done`).

### Изменено

- Для отслеживания переходов используется `chrome.storage.session` по ключу вкладка + пайплайн + stage; при закрытии вкладки эти записи удаляются.

## [1.0.0] — начальный релиз

Первая публичная версия: опрос вкладок MR / pipeline / job, уведомления и индикация при завершении выбранного stage CI в GitLab (с учётом fail-fast для `failed`, настроек для `skipped` / `canceled` и whitelist проектов).
